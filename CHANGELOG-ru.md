# Журнал изменений

Все значимые изменения проекта фиксируются в этом файле.

Формат основан на Keep a Changelog и адаптирован для данного репозитория.

## [1.1.0] - 2026-03-05

### Изменено

- Добавлены изменения совместимости для окружений с Python `3.8.x` и `urwid 2.0.1`.
- Переработано оформление заголовков `LineBox`, чтобы не использовать аргументы конструктора `urwid`, отсутствующие в старых версиях.
- В README обновлены требования: Python `3.8+` и `urwid>=2.0.1,<3.0`.
- Исправлено отображение ACL policy rules с табуляцией: табы нормализуются для корректного показа в терминале.
- В списке `Services` оставлены колонки `Name / Inst / Status`; колонка `Tags` удалена, её ширина перенесена в `Name`.
- Уточнено поведение фильтра `F7`:
  - в списке сервисов фильтрация только по имени сервиса;
  - в списках инстансов отдельный диалог с полями `Instance`, `Service`, `Address` и режимом `AND/OR`.
- Добавлено массовое выделение в списках `Services` и `Nodes`:
  - `Space` переключает выделение строки;
  - `F12` применяет выделение по regex-маске;
  - `Enter` открывает объединённый список инстансов для всех выделенных сервисов/нод.

## [1.0.0] - 2026-03-04

### Добавлено

- Первый публичный релиз `Consul Viewer TUI`
- Read-only terminal UI для Consul на базе `urwid`
- `Dashboard` со сводкой по кластеру и локальному агенту
- Раздел `Telemetry` на основе метрик агента в формате Prometheus
- Разделы `Services` и просмотр инстансов сервисов
- Разделы `Nodes` и просмотр инстансов на нодах
- KV browser с preview и полным viewer
- Список сессий и просмотр деталей
- ACL read-only разделы для токенов, политик, ролей и auth methods
- Текстовая фильтрация, status filter, структурный фильтр инстансов и сортировка для каждого вида списка
- Фоновая загрузка, TTL-кэш, stale-state и popup-диагностика
- Пользовательские инструкции, README-файлы и PlantUML-диаграммы архитектуры
